This project is JARVIS — an offline-first Windows desktop AI assistant.

Rules:

1. Stability first, features second

2. Never use placeholder logic or fake AI behavior

3. Fix root causes, not surface patches

4. Do not redesign UI unless explicitly requested

5. Wake word reliability is critical

6. Prefer SpeechRecognition fallback over unstable openwakeword

7. Avoid tensorflow and unnecessary heavy dependencies

8. Protect all microphone lifecycle handling carefully

9. Never break startup stability

10. Use production-safe exception handling

11. Preserve local-first architecture

12. Do not invent fake assistant intelligence

13. Memory must change future behavior, not just store notes

14. Decision engine must use clarification, not “No intent matched”

15. Learning engine must learn from correction history
